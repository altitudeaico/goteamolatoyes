Chapter III

The Hero's Journey

Four phases from idea to launch

1
IMAGINE
The Vision Phase

Before Tony Stark built the Iron Man suit, he had to imagine it. What do YOU want to build?

💡

Pro Tip: Start with "What if..."

What if families could play quiz games together from different countries? What if we could track who's winning in real-time?

Ask yourself:

  • What problem am I solving?
  • Who will use this?
  • What makes it fun or useful?
I want to build a [type of app] that lets [who] do [what]. For example: "I want to build a quiz game that lets my family play together from different houses, seeing each other's scores update live."
2
ARTICULATE
The Communication Phase

Doctor Strange doesn't just wave his hands randomly — his spells work because he speaks them precisely. The same applies to working with AI.

"The more specific you are, the more powerful Claude becomes."
— The First Rule of Prompting

Bad prompt:

Make me a quiz game

Good prompt:

Build me a multiplayer quiz game with these features: 1. PLAYERS - 4 teams can join using a game code - Each team picks from: Radlett 👑, New York 🗽, Middleton 🏰, Gravesend ⚓ 2. GAMEPLAY - 20 questions per game - 15 second timer per question - 4 multiple choice answers per question - 100 points for correct answers 3. REAL-TIME - All players see the same question at the same time - Scores update live for everyone - Show leaderboard every 5 questions 4. DESIGN - Dark theme with gold and purple colors - Fun sound effects - Mobile-friendly Use Supabase for the database and real-time sync.
✓ Create the HTML structure ✓ Add CSS styling with your colors ✓ Build the game logic in JavaScript ✓ Connect to Supabase for multiplayer ✓ Add sound effects and animations
🎯

The SECRET Formula

WHO + WHAT + HOW + DETAILS = Perfect Prompt

3
CREATE
The Building Phase

Now the magic happens! Claude will generate code. But here's the secret: building is iterative. You don't build Iron Man in one go — you build, test, improve, repeat.

🔨

The Build → Test → Improve Loop

Like Tony Stark going from Mark I to Mark 85, your app will evolve. Each version gets better!

Step 1: Get the basics working

Let's start simple. Create the home screen with: - A title "Family Quiz Night" - A "Host Game" button - A "Join Game" button - Use a dark background with golden text

Step 2: Add features one at a time

Great! Now add the team selection screen: - Show 4 team cards in a grid - Each card shows team name, emoji, and an image - When clicked, highlight the selected team - Add a "Join Game" button at the bottom

Step 3: Connect the pieces

Now connect it to Supabase so multiple players can join: - When host creates game, generate a 4-letter code - Save the game to Supabase - When player joins with code, add them to the game - Show all connected players in the lobby
🎮

Test After Each Step!

Before adding the next feature, make sure the current one works. Open your app, click buttons, try to break it!

4
REFINE
The Polish Phase

The difference between good and GREAT is in the details. This is where you make it shine! ✨

Fix bugs:

There's a bug: when the timer runs out, the beeping sound keeps playing forever. Can you fix it so all sounds stop when the quiz ends?

Add polish:

Make the leaderboard more exciting! Add: - Dramatic entrance animation (like a manga power-up) - Triumphant victory music - Speed lines in the background - Each team slides in one by one

Get feedback:

My family tested the game and said: 1. The text is too small on mobile 2. They want to see their team's image on the question screen 3. The countdown is hard to see Can you fix these issues?
"Genius is 1% inspiration and 99% perspiration."

Connect to Supabase 2. Insert data 3. Read data 4. Listen for real-time updates
🔧 Building Features
Add a New Screen
Add a new screen to my app called "[SCREEN_NAME]". It should include: - [ELEMENT 1] - [ELEMENT 2] - [ELEMENT 3] When the user clicks [BUTTON], go to [NEXT_SCREEN]. Style it to match the existing dark theme with [COLORS].
Add Sound Effects
Add sound effects to my app using the Web Audio API (no external files needed). I want sounds for: - [ACTION 1]: [DESCRIPTION] (e.g., "correct answer": happy chime going up) - [ACTION 2]: [DESCRIPTION] (e.g., "wrong answer": sad buzz going down) - [ACTION 3]: [DESCRIPTION] (e.g., "timer warning": urgent beeping) Also add a mute/unmute button.
Add Animations
Make my app more exciting with CSS animations: 1. When [ELEMENT] appears, it should [ANIMATION] (e.g., "slide in from the left and bounce") 2. When [EVENT] happens, add [EFFECT] (e.g., "confetti explosion when someone wins") 3. Add a subtle [BACKGROUND_EFFECT] (e.g., "twinkling stars in the background")
🔍 Debugging
Fix a Bug
I found a bug in my app: WHAT I EXPECTED: [DESCRIBE EXPECTED BEHAVIOR] WHAT ACTUALLY HAPPENED: [DESCRIBE THE BUG] STEPS TO REPRODUCE: 1. [STEP 1] 2. [STEP 2] 3. [STEP 3] Here's the relevant code: [PASTE CODE IF HELPFUL] Can you help me fix this?
Something Isn't Working
My [FEATURE] isn't working and I'm stuck. I'm trying to: [WHAT YOU WANT TO DO] I've tried: [WHAT YOU ALREADY ATTEMPTED] The error message (if any): [ERROR] Can you help me understand what's wrong and how to fix it?
✨ Making it Amazing
User Feedback
I tested my app with [WHO] and got this feedback: 1. [FEEDBACK 1] 2. [FEEDBACK 2] 3. [FEEDBACK 3] Can you help me improve the app based on this feedback?
Mobile Friendly
Make my app work better on mobile phones: 1. Text should be readable without zooming 2. Buttons should be easy to tap 3. Layout should adjust to smaller screens 4. Everything should fit without horizontal scrolling
Final Polish
My app is almost done! Help me with final polish: 1. Add a loading state so users know something is happening 2. Add friendly error messages if something goes wrong 3. Make sure the app looks good (consistent colors, spacing, fonts) 4. Add any small details that make it feel professional
For Parents & Guardians

Setup Guide

Everything you need to get your young coder started

  • 🤖

    Claude AI Account

    Each participant needs access to Claude. Options:
    • Claude Pro ($20/month) — Recommended for best experience
    • Free tier — Limited messages but works for learning
    → Sign up at claude.ai

  • 📦

    GitHub Account (Free)

    For storing code and hosting the live app. We'll provide a shared repository with access tokens so kids can push their code easily.
    → Sign up at github.com

  • 🗄️

    Supabase (Free Tier)

    The database for multiplayer features. We'll set up a shared project and provide credentials to all participants. No setup needed from parents!
    ✓ We handle this for the hackathon

  • 💻

    A Computer or Tablet

    Any device with a web browser works! Chrome, Safari, or Edge recommended. Laptop/desktop is easiest, but tablets work too.

  • 🌐

    Internet Connection

    Needed for Claude, GitHub, and testing multiplayer features. A stable connection is helpful but not critical.

  • 🎨

    Creativity & Patience

    The most important tools! Building things takes trial and error. Encourage your young coder to experiment, make mistakes, and keep trying. That's how real developers work!
